The two walls

Wall 1 — Duplicated override engine HIGH RISK

Override logic is written twice in two languages — TypeScript in the Api, PowerShell on the box — synced only by comments, using two different XPath engines. A selector can resolve to a different node on the box and write your edit to the wrong place, silently.

FIX → Workstream A · retire it (two-copy model)

Wall 2 — Un-abstracted UI render SCALING

Build-string → innerHTML → querySelectorAll → addEventListener, copy-pasted 13+ times. Two god-files: editor.js (1199 lines), map.js (2064). Fine at 6 tabs; breaks near 10.

FIX → Workstream B · json-ui + view split

Parallel work — json-ui

Another effort is building json-ui: a generic, no-build JSON-editor package (mountEditor(el, value, {onChange}), host owns the save). It replaces the bespoke field/tree editors — the densest slice of Wall 2 — and it is the two-copy editor: edit the whole doc, host saves the whole file, no delta. It does not cover the general view chrome (Wall 2b), which stays a separate call. Full alignment in UI-ABSTRACTION-CONTRACT.md.
